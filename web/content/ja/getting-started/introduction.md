# はじめに

oh-my-agent は Antigravity IDE 向けのマルチエージェントオーケストレーターです。リクエストを専門スキルにルーティングし、Serena メモリを通じてエージェントを連携させます。

## 提供される機能

- リクエスト駆動のスキルルーティング
- 計画/レビュー/デバッグのためのワークフローベース実行
- エージェントの並列実行のための CLI オーケストレーション
- セッション監視のためのリアルタイムダッシュボード

## エージェントの役割

| エージェント | 責務 |
|---|---|
| oma-coordination | 複雑なマルチドメインプロジェクトの調整 |
| oma-pm | 計画とアーキテクチャ分解 |
| oma-frontend | React/Next.js の実装 |
| oma-backend | バックエンドAPI実装 (Python, Node.js, Rust, ...) |
| oma-mobile | Flutter/モバイルの実装 |
| oma-qa | セキュリティ/パフォーマンス/アクセシビリティレビュー |
| oma-debug | 根本原因分析とリグレッションセーフな修正 |
| oma-brainstorm | 設計優先のアイデア創出とコンセプト探索 |
| oma-db | データベースモデリング、スキーマ設計、クエリチューニング |
| oma-dev-workflow | 開発ワークフロー最適化と CI/CD |
| oma-tf-infra | Terraform インフラストラクチャ・アズ・コード プロビジョニング |
| oma-translator | コンテキスト対応多言語翻訳 |
| oma-orchestrator | CLI ベースのサブエージェントオーケストレーション |
| oma-commit | コンベンショナルコミットワークフロー |

## プロジェクト構成

- `.agents/skills/`: スキル定義とリソース
- `.agents/workflows/`: 明示的なワークフローコマンド
- `.serena/memories/`: ランタイムオーケストレーション状態
- `cli/cli.ts`: コマンドインターフェースの正式なソース

## 段階的開示

1. リクエストの意図を特定
2. 必要なスキルリソースのみを読み込み
3. 専門エージェントで実行
4. QA/デバッグループで検証と反復
