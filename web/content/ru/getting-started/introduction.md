# Введение

oh-my-agent — это мультиагентный оркестратор для Antigravity IDE. Он маршрутизирует запросы к специализированным навыкам и координирует агентов через память Serena.

## Что вы получаете

- Маршрутизация навыков на основе запроса
- Выполнение на основе рабочих процессов для планирования/ревью/отладки
- CLI-оркестрация для параллельного запуска агентов
- Панели мониторинга в реальном времени для отслеживания сессий

## Роли агентов

| Агент | Ответственность |
|---|---|
| oma-coordination | Координация сложных мультидоменных проектов |
| oma-pm | Планирование и архитектурная декомпозиция |
| oma-frontend | Реализация на React/Next.js |
| oma-backend | Реализация бэкенд API (Python, Node.js, Rust, ...) |
| oma-mobile | Реализация на Flutter/мобильная разработка |
| oma-qa | Ревью безопасности/производительности/доступности |
| oma-debug | Анализ корневых причин и безопасные исправления без регрессий |
| oma-brainstorm | Дизайн-ориентированная идеация и исследование концепций |
| oma-db | Моделирование баз данных, проектирование схем и оптимизация запросов |
| oma-dev-workflow | Оптимизация рабочих процессов разработки и CI/CD |
| oma-tf-infra | Provisioning инфраструктуры как кода с Terraform |
| oma-translator | Контекстно-зависимый многоязычный перевод |
| oma-orchestrator | CLI-оркестрация субагентов |
| oma-commit | Рабочий процесс конвенциональных коммитов |

## Структура проекта

- `.agents/skills/`: определения навыков и ресурсы
- `.agents/workflows/`: явные команды рабочих процессов
- `.serena/memories/`: состояние оркестрации во время выполнения
- `cli/cli.ts`: источник истины для интерфейсов команд

## Прогрессивное раскрытие

1. Определение намерения запроса
2. Загрузка только необходимых ресурсов навыка
3. Выполнение специализированными агентами
4. Верификация и итерация через циклы QA/отладки
